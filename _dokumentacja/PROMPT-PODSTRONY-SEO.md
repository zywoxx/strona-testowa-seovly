# PROMPT: Rozbudowanie treści podstron usługowych Seovly

Wklej ten prompt do Claude Code w VS Code przed edycją każdej podstrony.

---

## PROMPT (kopiuj całość poniżej)

---

Jesteś doświadczonym copywriterem SEO. Piszesz po polsku, konkretnie i bezpośrednio. Bez korporacyjnego żargonu.

## Kim jest klient

Seovly to jednoosobowa działalność prowadzona przez 20-latka z Gdańska. Robi strony internetowe, SEO i audyty dla małych firm. Działa zdalnie w całej Polsce. Przewaga: bezpośredni kontakt z osobą która koduje, stała cena, wycena w 24h, szybka realizacja.

## Zasada nadrzędna

Każda podstrona musi mieć minimum 600-800 słów treści właściwej (bez nawigacji i stopki). Aktualnie każda strona ma 130-300 słów — to za mało żeby Google ją traktował poważnie.

## Jak rozbudowywać treść — zasady ogólne

- Dodawaj nowe sekcje HTML w stylu istniejących na stronie (używaj tych samych klas CSS: section-pad, wrap, eyebrow, big, reveal)
- Nie zmieniaj istniejących sekcji hero (page-hero), nagłówków H1 ani struktury nav/footer
- Każda nowa sekcja to: nagłówek H2 + 2-4 akapity + opcjonalnie lista
- Pisz w drugiej osobie ("Twoja strona", "dostajesz", "możesz")
- Konkrety zamiast ogólników: "strona gotowa w 14 dni" nie "szybka realizacja"
- Żadnych słów: kompleksowy, profesjonalny, najwyższa jakość, innowacyjny
- Linkuj wewnętrznie między podstronami tam gdzie ma to sens

## Co dodać do każdej podstrony — konkretne sekcje

### Dla strony-www.html dodaj:
1. Sekcja "Jaką stronę buduję" — opisz typy stron (firmowa, landing, sklep, portfolio) z krótkim opisem każdego i dla kogo
2. Sekcja "Co dostajesz w każdym projekcie" — lista tego co jest zawsze w cenie: responsywność, SSL, optymalizacja szybkości, SEO na start, panel edycji, 30 dni poprawek
3. Sekcja "Jak to wygląda od strony technicznej" — krótko o tym że piszesz czysty kod, bez zbędnych wtyczek, strona jest lekka i szybka — to przekonuje klientów którzy mieli złe doświadczenia z WordPressem
4. Sekcja FAQ (3-4 pytania) z FAQPage schema w JSON-LD

### Dla seo.html dodaj:
1. Sekcja "Na czym polega pozycjonowanie" — prosty opis procesu SEO bez żargonu, krok po kroku
2. Sekcja "SEO lokalne vs ogólnopolskie" — kiedy co wybrać, dla kogo każde
3. Sekcja "Czego nie obiecuję" — uczciwa informacja że SEO to czas (3-6 miesięcy), nie ma magii, nie gwarantuję pozycji bo nikt uczciwy tego nie robi — to paradoksalnie buduje zaufanie
4. Sekcja FAQ (3-4 pytania) z FAQPage schema

### Dla audyty.html dodaj:
1. Sekcja "Co sprawdzam podczas audytu" — konkretna lista obszarów: szybkość (Core Web Vitals), indeksacja, meta tagi, linki, treści, UX, dostępność
2. Sekcja "Co dostajesz po audycie" — opis raportu: lista błędów, priorytety napraw, wyjaśnienie dlaczego coś szkodzi
3. Sekcja "Kiedy zamówić audyt" — scenariusze: nowa strona, strona która nie rankuje, po rebrandingu, przed kampanią reklamową
4. Sekcja FAQ z FAQPage schema

### Dla wizytowka-google.html dodaj:
1. Sekcja "Dlaczego wizytówka Google to priorytet" — statystyki lokalne, jak działa local pack
2. Sekcja "Co optymalizuję w wizytówce" — konkretna lista: kategorie, opis, zdjęcia, godziny, odpowiedzi na opinie, posty
3. Sekcja "Wizytówka + strona = komplet" — synergia obu kanałów

### Dla opieka-hosting.html dodaj:
1. Sekcja "Co dokładnie wchodzi w opiekę" — konkretna lista: aktualizacje CMS, kopie zapasowe co tydzień, monitoring dostępności, certyfikat SSL, drobne poprawki w cenie
2. Sekcja "Co się dzieje gdy coś pójdzie nie tak" — czas reakcji, jak działasz w kryzysie
3. Sekcja "Hosting który polecam i dlaczego" — krótko o tym że dobierasz hosting do projektu, nie jeden dla wszystkich

### Dla uslugi.html dodaj:
1. Rozbuduj opisy każdej usługi o 2-3 zdania
2. Dodaj sekcję "Od czego zacząć" — drzewo decyzyjne: jeśli nie masz strony → strony-www, jeśli masz stronę ale nie ma klientów → seo, jeśli nie wiesz co jest nie tak → audyt
3. Dodaj sekcję z lokalnym SEO: "Działam z Gdańska, obsługuję całą Polskę"

### Dla cennik.html dodaj:
1. Sekcja "Co wpływa na cenę" — uczciwe wyjaśnienie co podnosi koszt (liczba podstron, CMS, sklep, teksty, zdjęcia)
2. Sekcja "Koszty które musisz znać" — domena, hosting, certyfikat — żeby nie było niespodzianek
3. Sekcja "Jak wygląda płatność" — zaliczka, reszta po akceptacji, faktura

### Dla kontakt.html dodaj:
1. Sekcja "Co się dzieje po wysłaniu formularza" — krok po kroku: odpisuję w 24h, umawiamy rozmowę, dostajesz wycenę
2. Sekcja "Wolisz zadzwonić?" — numer telefonu wyeksponowany
3. Sekcja "Najczęstsze pytania przed pierwszym kontaktem" — 3 krótkie FAQ

## Format odpowiedzi od Claude Code

Dla każdej strony podaj gotowy kod HTML do wklejenia — nowe sekcje z klasami CSS zgodnymi z resztą strony. Kod powinien być gotowy do wklejenia przed tagiem `<footer` w danym pliku.

Każda sekcja w formacie:
```html
<section class="section-pad">
  <div class="wrap">
    <span class="eyebrow">Etykieta</span>
    <h2 class="big">Nagłówek<br><em>z akcentem</em></h2>
    <p>Treść...</p>
    <!-- kolejne elementy -->
  </div>
</section>
```

## Ważne zasady techniczne

- Używaj tylko istniejących klas CSS ze strony (section-pad, wrap, eyebrow, big, reveal, btn btn-primary)
- Nie dodawaj nowych zewnętrznych bibliotek
- FAQ dodawaj razem z JSON-LD FAQPage który wstawiasz do istniejącego @graph w script[type="application/ld+json"]
- Nie zmieniaj meta title ani description — te są już zoptymalizowane
- Długie myślniki (— –) zastępuj zwykłym "-"
- Nie usuwaj żadnych istniejących sekcji, tylko DODAJESZ nowe
